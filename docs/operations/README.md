## 安全规则

- Git 中只记录账号用途、账号名、邮箱和密码占位符。
- 密码统一写为 `<在密码管理器中填写>`。
- API Key、数据库密码、SMTP 密码和 DNS Token 统一写为 `<SECRET:用途>`。
- Railway、Vercel、AgentMail、GitHub 和域名商的真实密钥不得写入本目录。
- 根管理员、GitHub 所有者和域名所有者必须启用二次认证，并保存恢复码。

## 更新责任

- 每次生产部署后更新版本、提交号、部署时间和验证结果。
- 每次更换域名、邮箱、上游渠道或数据库后更新服务关系表。
- 每次事故后补充“故障与恢复记录”，写明原因、修复和防复发措施。
