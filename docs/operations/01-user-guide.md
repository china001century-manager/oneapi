# WBoke 用户使用手册

适用对象：购买并使用 WBoke AI API 的普通用户。

## 1. 访问与注册

正式入口：`https://www.wboke.com`

当前状态：正式域名已经切换并可访问，但注册邮件、找回密码和 Windows 安装包仍处于试运行验收阶段，不应对外宣称全部功能已正式上线。

试运行备用入口：`https://wbokedesktop-production.up.railway.app`

注册流程：

1. 打开官网，进入注册页面。
2. 填写用户名、个人邮箱和密码。
3. 点击“发送验证码”。
4. 在邮箱中查收来自 `wboke-5247@agentmail.to` 的验证码。
5. 填写验证码并完成注册。
6. 邀请码不是必填项。

如果没有收到验证码：

- 检查垃圾邮件和订阅邮件目录。
- 确认邮箱地址没有多余空格或拼写错误。
- 等待 60 秒后再发送一次，不要连续点击。
- 向管理员提供注册时间、邮箱和页面错误信息，不要提供密码。

当前邮件仍处于试运行观察期。页面提示发送成功不代表邮箱服务商已经完成最终投递；如果 5 分钟仍未收到，应停止重复发送并联系管理员查投递记录。

## 2. 购买与兑换

WBoke 首版不在站内直接收款。用户通过公司批准的链小铺商品链接购买兑换码，再回到 WBoke 手动兑换。

计划面额：`1`、`5`、`10`、`20`、`50`、`100` 元。

链小铺商品入口：

- `<LIANXIAOPU_PRODUCT_URL_1>`
- `<LIANXIAOPU_PRODUCT_URL_2>`
- `<按面额继续补充>`

兑换流程：

1. 在链小铺完成购买并取得兑换码。
2. 登录 WBoke 控制台。
3. 进入“钱包/兑换码”。
4. 输入兑换码并确认。
5. 核对余额和兑换记录。

安全提示：管理员不会索要用户密码、API Key 或邮箱验证码。兑换码只能使用一次，不要公开发送。

## 3. 创建 API Key

1. 登录后进入“API 密钥”。
2. 创建一个用途明确的 Key，例如 `Office-PC-Codex`。
3. 根据需要设置额度、模型和有效期。
4. Key 只显示一次时应立即保存到密码管理器。
5. 不同电脑或工具使用不同 Key，方便单独撤销。

正式 API Base URL：`https://www.wboke.com/v1`

试运行备用 Base URL：`https://wbokedesktop-production.up.railway.app/v1`

不要把 Key 发到聊天群、截图、工单或公开代码仓库。

## 4. Windows 客户端

客户端目标流程：

1. 从官网下载安装包。
2. 使用 WBoke 网站账号登录客户端。
3. 客户端从 New API 获取或创建当前用户自己的 API Key。
4. 用户选择要配置的工具和协议。
5. 客户端将 Base URL 和用户 Key 写入对应工具的配置文件。
6. 客户端显示写入位置和验证结果。

客户端不会获得管理员凭据，也不会把所有用户共用一个后台 Key。

## 5. 常见配置

OpenAI 兼容工具：

```text
Base URL: https://www.wboke.com/v1
API Key: <USER_API_KEY>
```

Anthropic、Gemini 和其他协议的具体环境变量应以客户端内的协议选择为准。不要把已遮罩的 Key（例如 `sk-****`）当作真实 Key 写入配置。

## 6. 故障排查

| 现象 | 用户可检查 | 管理员需检查 |
|---|---|---|
| 验证码未收到 | 邮箱地址、垃圾邮件、等待 60 秒 | New API 日志、Relay 日志、AgentMail 发送/退信 |
| 登录失败 | 邮箱/用户名、密码、验证码 | 用户状态、登录限流、系统时间 |
| 重定向次数过多 | 使用无痕窗口，记录完整 URL | 已修复空路径 rewrite；继续检查 ServerAddress、Cookie 与代理跳转链 |
| API 返回 401 | Key 是否完整、是否被撤销 | Token 状态与权限 |
| API 返回 402 | 余额是否足够 | 兑换记录与计费日志 |
| API 返回 429 | 等待后重试 | 用户/分组/上游限流 |
| 模型不存在 | 使用控制台显示的模型名 | 渠道模型映射与价格配置 |
| 客户端写入失败 | 关闭占用配置文件的程序 | 路径适配、权限、备份与回滚日志 |
